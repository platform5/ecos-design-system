Understanding the Ecos project

The main goal of this project (at the moment) is to experiment FAST and try to build things with it.

What kind of experiment ?

  • Experimenting extending the FAST Frame design system. If this works well it should enable the project to provide a Design System of its own.
  • Experimenting how components work together when used in common use cases. The idea is to put together a homepage, a doc page, a blog page and an admin dashboard.
  • Playing with design tokens on the fly and see how they affect the overall result.

How is the project organized ?

The design system

The main piece of the project is the design system itself. It is found in the /src/ecos-design-system folder. This folder can be extracted and published on its own.

# The following command builds the ecos-design-system as a standalone module ready to be published on NPM npm run build

The testing app

In order to play with the design system there is an Aurelia 2 application in the /src folder.

# The following command starts the app on localhost:9000 npm run start # The following command builds the app npm run build-app
# This folder contains app-specific components. Either ECOS components or AU2 custom elements; /src/custom-elements # Codebase for the whole design system experiment /src/ecos-design-system # This folder contains the routes modules loaded by the AU2 app /src/routes # Some app-specific CSS files /src/*.css # ecos-app.* are the main application files /src/*.css # Entry point for the AU2 application /src/main.ts
